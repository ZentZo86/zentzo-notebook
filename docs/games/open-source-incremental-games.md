# 开源增量游戏仓库精选

> 🎮 游戏调研 | 2026-02-04

增量游戏（Incremental Games）是一种以数值增长和自动化为核心玩法的游戏类型。本文档整理了 GitHub 上最具代表性的开源增量游戏项目，涵盖从经典神作到现代框架、从完整游戏到工具库的全方位资源。这些项目不仅是优秀的游戏作品，更是学习游戏开发、数值系统设计和前端技术的宝贵资源。

---

## 顶流神作

### 1. Antimatter Dimensions（反物质维度）

**GitHub**: [IvarK/AntimatterDimensionsSourceCode](https://github.com/IvarK/AntimatterDimensionsSourceCode)  
⭐ **306** | 🔱 **232** | 💾 **12,162 commits** | 👥 **50+ contributors**

**游戏简介**

Antimatter Dimensions 是增量游戏领域的绝对顶流神作，以其复杂的数值系统和天文数字处理而闻名。游戏从购买第一个反物质维度开始，逐步解锁更高维度、时间膨胀、现实重置等多层次系统，数值可以达到 e1000 甚至更高的量级。

**技术栈**

- **JavaScript**: 58.1% - 核心游戏逻辑
- **Vue.js**: 34.5% - 用户界面框架
- **CSS**: 7.2% - 样式设计
- **HTML**: 0.2% - 页面结构

**作为独立游戏制作者，你能学到什么？**

**天文数字处理技术**：项目实现了一套完整的大数运算库，能够处理超过 JavaScript Number 类型极限的数值。这对于任何需要处理指数增长的游戏都是必备技能。你将学会如何使用科学计数法表示、如何实现大数的加减乘除运算，以及如何优化性能避免计算卡顿。

**多层次进度系统设计**：游戏包含维度购买、牺牲、挑战、无限、永恒、现实等多个相互嵌套的进度层级。你将学会如何设计让玩家保持长期动力的进度曲线，如何平衡不同系统之间的关系，以及如何通过"重置获得加成"的机制让玩家愿意放弃当前进度。

**Vue.js 在游戏开发中的应用**：项目展示了如何使用 Vue.js 构建响应式的游戏界面，如何管理复杂的游戏状态，以及如何优化渲染性能。代码量超过 12,000 次提交，是学习 Vue.js 在大型项目中应用的优秀案例。

**模块化代码架构**：游戏的各个系统（维度、牺牲、挑战等）都被组织成独立的模块，便于维护和扩展。你将学会如何设计清晰的代码结构，避免意大利面条式代码。

### 2. Evolve（进化）

**GitHub**: [pmotschmann/Evolve](https://github.com/pmotschmann/Evolve)  
**在线游戏**: [https://pmotschmann.github.io/Evolve/](https://pmotschmann.github.io/Evolve/)  
⭐ **1,100** | 🔱 **503** | 💾 **4,686 commits** | 👥 **107 contributors**

**游戏简介**

Evolve 是 GitHub 上 star 数最高的开源增量游戏，讲述了从原始生命进化到太空文明的史诗故事。游戏结合了点击器和放置游戏的元素，包含大量微观管理。玩家需要管理资源、科技、人口、建筑等，逐步将文明从单细胞生物发展到跨星系帝国。

**技术栈**

- **JavaScript**: 97.3% - 游戏核心
- **Less**: 2.5% - CSS 预处理器
- **许可证**: MPL-2.0

**作为独立游戏制作者，你能学到什么？**

**文明进化系统设计**：游戏包含从原始生命到太空时代的完整科技树，涵盖生物进化、部落发展、工业革命、太空探索等多个阶段。你将学会如何设计长期的进度系统，如何让每个阶段都有独特的玩法，以及如何通过科技解锁逐步引入新机制。

**资源管理与生产链**：游戏包含数十种资源和复杂的生产关系。你将学会如何设计资源转化系统，如何平衡资源的产出和消耗，以及如何通过瓶颈资源创造策略深度。

**多语言国际化**：项目支持包括中文在内的多种语言，使用完善的本地化系统。你将学会如何组织翻译文件，如何实现语言切换，以及如何处理不同语言的文本长度差异。

**社区驱动开发**：拥有 107 位贡献者，是社区驱动开发的典范。你将学会如何管理开源项目，如何处理 Pull Request，以及如何与社区协作。

**构建与部署系统**：项目包含完整的构建系统，支持 Docker 部署和 GitHub Pages 自动发布。你将学会如何搭建自动化的开发流程。

### 3. Kittens Game（猫咪游戏）

**GitHub**: [nuclear-unicorn/kittensgame](https://github.com/nuclear-unicorn/kittensgame)  
**在线游戏**: [https://kittensgame.com](https://kittensgame.com)  
⭐ **62** | 🔱 **38** | 💾 **7,303 commits** | 👥 **69 contributors**

**游戏简介**

Kittens Game 是资源管理类增量游戏的天花板之作，被 Reddit 社区评价为"最好的增量游戏之一"。游戏以猫咪村庄为背景，玩家需要管理资源、建造建筑、研究科技，逐步发展文明。游戏的深度和复杂度远超表面，包含天文观测、时间旅行、宗教系统等多个高级机制。

**技术栈**

- **JavaScript**: 53.9% - 游戏核心（不使用 ES6）
- **CSS**: 42.9% - 界面样式
- **HTML**: 3.2% - 页面结构
- **框架**: Dojo (pre-1.7)

**作为独立游戏制作者，你能学到什么？**

**极致的浏览器兼容性**：项目支持包括 IE6 在内的古老浏览器，甚至可能在 Netscape 上运行。代码严格避免使用 ES6 语法，支持约 20,000 种不同的 Android 设备。你将学会如何编写兼容性最强的代码，如何处理不同浏览器的差异，以及如何在没有现代 JavaScript 特性的情况下实现复杂功能。

**资源管理系统的艺术**：游戏包含数十种资源和复杂的生产链，每个资源都有多种用途和获取方式。你将学会如何设计多层次的资源转化系统，如何通过资源瓶颈引导玩家决策，以及如何平衡短期收益和长期投资。

**长期项目维护经验**：项目历经多年持续更新，提交数超过 7,300 次。你将学会如何保持代码的可维护性，如何在不破坏现有功能的情况下添加新内容，以及如何处理技术债务。

**严谨的开发哲学**：项目文档明确指出"仓库中没有无用代码"、"永远不要削弱（nerf）任何东西"等设计原则。你将学会如何建立自己的设计哲学，如何在添加新功能时保持游戏的一致性。

**测试与质量保证**：项目包含完整的测试套件，确保代码质量。你将学会如何为游戏编写测试，如何进行回归测试，以及如何避免引入 bug。

## 现代框架与工具

### 4. The Modding Tree（TMT）

**GitHub**: [Acamaeda/The-Modding-Tree](https://github.com/Acamaeda/The-Modding-Tree)  
⭐ **117** | 🔱 **1,600+** | 💾 **578 commits**

**项目简介**

The Modding Tree 是一个"神级"增量游戏开发框架，现在 GitHub 上大部分爆火的小型增量游戏都是基于这个框架改造而成。它极大地降低了增量游戏的开发门槛，使得开发者可以专注于游戏设计而非技术实现。

**技术栈**

- **JavaScript**: 90.2% - 框架核心
- **HTML**: 5.0% - 模板结构
- **CSS**: 4.8% - 样式系统
- **基于**: Jacorb90/Prestige-Tree

**作为独立游戏制作者，你能学到什么？**

**快速原型开发**：TMT 提供了一套完整的增量游戏模板，你只需要定义游戏的数值和规则，框架会自动处理界面渲染、数据存储、保存/加载等基础功能。你将学会如何使用框架快速验证游戏创意，如何在几天内搭建出可玩的原型。

**模块化游戏设计**：框架采用"层（Layer）"的概念组织游戏内容，每个层可以包含升级、挑战、成就、可购买项等元素。你将学会如何将游戏拆分成独立的模块，如何设计可复用的游戏机制。

**声明式编程思想**：使用 TMT 开发游戏时，你只需要声明"有什么"而非"怎么做"。你将学会声明式编程的思想，理解数据驱动设计的优势。

**社区生态系统**：拥有超过 1,600 个 Fork，说明有大量开发者基于这个框架创建自己的游戏。你将学会如何参与开源社区，如何从其他人的项目中学习，以及如何为框架做贡献。

**游戏设计模式**：通过研究 TMT 的实现，你将理解增量游戏的核心设计模式，包括重置系统、升级系统、挑战系统等，这些模式可以应用到任何增量游戏中。

### 6. Level 13（13 层）

**GitHub**: [nroutasuo/level13](https://github.com/nroutasuo/level13)  
**在线游戏**: [https://nroutasuo.github.io/level13/](https://nroutasuo.github.io/level13/)  
⭐ **248** | 🔱 **84** | 💾 **3,534 commits** | 📦 **v0.6.3**

**游戏简介**

Level 13 是一个基于文本的增量科幻浏览器冒险游戏，玩家必须在黑暗、衰败的城市中生存，重新发现旧技术和新技术，并重建已经崩溃的文明。游戏受《A Dark Room》启发，同时借鉴了 Kittens Game、Shark Game 等经典作品。

**技术栈**

- **JavaScript** - 游戏核心
- **jQuery** - DOM 操作
- **Require.js** - 模块加载
- **Ash.js** - 实体组件系统
- **许可证**: MIT

**作为独立游戏制作者，你能学到什么？**

**实体-组件-系统（ECS）架构**：项目使用 Ash.js 实现了完整的 ECS 架构，这是游戏开发中非常重要的设计模式。你将学会如何将游戏对象拆分成组件，如何通过系统处理游戏逻辑，以及如何实现高度解耦的代码结构。

**程序化生成技术**：游戏使用种子值生成唯一的世界，包含 WorldGenerator、LevelGenerator、StructureGenerator 等模块。你将学会如何实现程序化生成，如何使用种子保证可重现性，以及如何设计生成算法。

**文本冒险游戏设计**：游戏展示了如何用纯文本创造沉浸感，如何通过描述激发玩家想象。你将学会如何设计文本界面，如何平衡信息量和可读性，以及如何用文字讲故事。

**生存与探索机制**：游戏包含基地建设、资源管理、探索、战斗等多个系统。你将学会如何设计生存游戏的核心循环，如何平衡风险和收益，以及如何通过探索驱动玩家前进。

**技术树设计**：游戏包含逐步解锁新游戏方面的技术树。你将学会如何设计技术树，如何控制解锁节奏，以及如何通过科技引入新玩法。

**游戏简介**

Progress Knight 是一个用纯 JavaScript 构建的放置游戏，玩家通过完成任务和升级来推进角色发展。游戏以简洁的代码和清晰的逻辑著称，非常适合初学者学习。

## 数学库与工具

### 9. break_infinity.js

**GitHub**: [Patashu/break_infinity.js](https://github.com/Patashu/break_infinity.js)  
⭐ **234** | 📦 **TypeScript**

**库简介**

break_infinity.js 是专为增量游戏设计的数学库，用于处理超大数字（最大可达 1e(9e15)），是 decimal.js 的高性能替代品。该库优先考虑速度而非精度，非常适合需要频繁进行大数运算的增量游戏。

**作为独立游戏制作者，你能学到什么？**

**大数运算原理**：你将学会如何表示超过 JavaScript Number 类型极限的数字，如何实现科学计数法的加减乘除运算，以及如何优化性能。

**性能优化技巧**：库的设计目标是速度，你将学会如何在精度和性能之间取舍，如何避免不必要的计算，以及如何使用缓存提升性能。

**TypeScript 开发**：项目使用 TypeScript 编写，提供完整的类型定义。你将学会如何使用 TypeScript 开发库，如何编写类型安全的代码。

**库的设计与发布**：你将学会如何设计一个可复用的库，如何编写文档，如何发布到 npm，以及如何维护开源项目。

### 10. break_eternity.js

**GitHub**: [Patashu/break_eternity.js](https://github.com/Patashu/break_eternity.js)  
⭐ **174** | 📦 **JavaScript**

**库简介**

break_eternity.js 是 break_infinity.js 的续作，可以表示高达 10^^1e308 的数字和小至 10^-10^^1e308 的数字。专为需要处理极大数字的增量游戏设计，支持四则运算、对数、幂运算等操作。

**作为独立游戏制作者，你能学到什么？**

**极限数值处理**：你将学会如何处理超越想象的巨大数字，如何实现多层指数运算，以及如何在极端情况下保持数值稳定。

**数学算法实现**：库实现了复杂的数学运算，你将学会如何将数学公式转化为代码，如何处理边界情况，以及如何验证算法正确性。

**库的演进与维护**：通过对比 break_infinity 和 break_eternity，你将学会如何改进现有库，如何在保持兼容性的同时扩展功能。

## 学习路径建议

对于想要深入学习增量游戏开发的开发者，建议按以下路径进行：

### 第一阶段：快速入门（1-2 周）

**目标**：理解增量游戏的基本概念，搭建第一个原型。

**推荐项目**：
1. **Progress Knight** - 阅读源码，理解基本结构
2. **The Modding Tree** - 使用框架快速搭建原型

**学习重点**：
- 增量游戏的核心循环（点击 → 获得资源 → 购买升级 → 自动化）
- 如何保存和加载游戏进度
- 如何实现基本的数值增长

### 第二阶段：深入理解（2-4 周）

**目标**：学习现代增量游戏的设计模式和代码组织方式。

**推荐项目**：
1. **Incremental Mass Rewritten** - 研究现代设计
2. **break_infinity.js** - 学习大数处理

**学习重点**：
- 如何处理超大数值
- 如何设计重置系统（Prestige）
- 如何优化用户体验
- 如何使用 CSS 创建吸引人的界面

### 第三阶段：系统设计（4-8 周）

**目标**：学习如何设计复杂的游戏系统。

**推荐项目**：
1. **Kittens Game** - 深入研究资源管理
2. **Evolve** - 学习文明进化系统

**学习重点**：
- 如何设计复杂的资源转化链
- 如何平衡多个系统之间的关系
- 如何设计长期的进度曲线
- 如何处理技术债务

### 第四阶段：高级技术（8+ 周）

**目标**：掌握增量游戏开发的高级技术。

**推荐项目**：
1. **Antimatter Dimensions** - 学习复杂系统架构
2. **Level 13** - 学习 ECS 架构和程序化生成

**学习重点**：
- 如何处理极端情况和优化性能
- 如何实现实体组件系统
- 如何设计程序化生成算法
- 如何维护大型项目

### 第五阶段：创新与发布（持续）

**目标**：创造自己的增量游戏并发布。

**行动建议**：
1. 设计独特的游戏主题和机制
2. 使用学到的技术实现原型
3. 收集玩家反馈并迭代
4. 发布到 GitHub 和游戏平台
5. 参与社区，分享经验
