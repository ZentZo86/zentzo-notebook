# 开源增量游戏仓库精选

> 🎮 游戏调研 | 2026-02-04

增量游戏（Incremental Games）是一种以数值增长和自动化为核心玩法的游戏类型。本文档整理了四个在 GitHub 上具有代表性的开源增量游戏项目，涵盖了从经典神作到现代框架的不同类型。这些项目不仅是优秀的游戏作品，更是学习游戏开发、数值系统设计和前端技术的宝贵资源。

---

## 1. Antimatter Dimensions（反物质维度）

**Antimatter Dimensions** 是增量游戏领域的绝对顶流神作，以其复杂的数值系统和天文数字处理而闻名。该游戏展示了如何在浏览器环境中处理超大数值运算，以及如何构建多层次的进度系统。

### 项目信息

| 项目 | 信息 |
|------|------|
| **GitHub 仓库** | [IvarK/AntimatterDimensionsSourceCode](https://github.com/IvarK/AntimatterDimensionsSourceCode) |
| **Stars** | 306 ⭐ |
| **Forks** | 232 🔱 |
| **贡献者** | 50+ 人 |
| **提交数** | 12,162 次 |
| **许可证** | MIT |

### 技术栈

该项目采用现代前端技术栈构建，技术组成如下：

- **JavaScript**: 58.1% - 核心游戏逻辑
- **Vue.js**: 34.5% - 用户界面框架
- **CSS**: 7.2% - 样式设计
- **HTML**: 0.2% - 页面结构

### 核心特点

**天文数字处理**：游戏中的数值可以达到 e1000 甚至更高的量级，项目实现了一套完整的大数运算库，这对于任何需要处理超大数值的应用都具有参考价值。

**复杂系统架构**：游戏包含多个相互关联的系统，包括维度购买、时间膨胀、现实重置等，展示了如何设计和管理复杂的游戏状态机。

**Vue.js 实践**：项目使用 Vue.js 构建响应式界面，代码量超过 12,000 次提交，是学习 Vue.js 在大型项目中应用的优秀案例。

### 适用场景

- 学习如何处理超大数值运算
- 研究复杂游戏系统的架构设计
- 了解 Vue.js 在游戏开发中的应用
- 参考增量游戏的数值平衡设计

## 2. Kittens Game（猫咪游戏）

**Kittens Game** 是资源管理类增量游戏的天花板之作，以其扎实的代码逻辑和庞大的游戏内容而著称。该项目的代码量巨大，但逻辑清晰，是学习资源管理系统设计的绝佳范例。

| **许可证** | 未明确标注 |
| **官方网站** | [kittensgame.com](https://kittensgame.com) |

### 技术栈

项目采用经典的前端技术组合：

- **JavaScript**: 53.9% - 游戏核心逻辑
- **CSS**: 42.9% - 界面样式
- **HTML**: 3.2% - 页面结构

### 核心特点

**极致的浏览器兼容性**：项目的一大特色是支持超广泛的浏览器版本，包括 IE6 在内的古老浏览器。代码严格避免使用 ES6 语法，确保在各种设备和浏览器上都能流畅运行。

**资源管理系统**：游戏包含数十种资源类型和复杂的生产链，展示了如何设计和平衡多层次的资源转化系统。

**长期维护**：项目历经多年持续更新，提交数超过 7,300 次，代码结构经过长期打磨，具有很高的稳定性和可维护性。

**严格的代码规范**：项目文档明确指出"仓库中没有无用代码"，强调代码的每一行都有其存在的意义，这种严谨的开发态度值得学习。

### 适用场景

- 学习资源管理系统的设计
- 研究如何实现广泛的浏览器兼容性
- 了解长期维护项目的代码组织方式
- 参考复杂游戏逻辑的实现

## 3. The Modding Tree（TMT）

**The Modding Tree** 是一个"神级"增量游戏开发框架，现在 GitHub 上大部分爆火的小型增量游戏都是基于这个框架改造而成。它极大地降低了增量游戏的开发门槛，使得开发者可以快速实现自己的创意。

### 项目信息

| 项目 | 信息 |
|------|------|
| **GitHub 仓库** | [Acamaeda/The-Modding-Tree](https://github.com/Acamaeda/The-Modding-Tree) |
| **Stars** | 117 ⭐ |
| **Forks** | 1,600+ 🔱 |
| **提交数** | 578 次 |
| **许可证** | 未明确标注 |
| **基于项目** | Jacorb90/Prestige-Tree |

### 技术栈

项目以 JavaScript 为主导：

- **JavaScript**: 90.2% - 框架核心
- **HTML**: 5.0% - 模板结构
- **CSS**: 4.8% - 样式系统

### 核心特点

**快速开发框架**：TMT 提供了一套完整的增量游戏开发模板，开发者只需要定义游戏的数值和规则，框架会自动处理界面渲染、数据存储等基础功能。

**高度模块化**：框架采用层（Layer）的概念组织游戏内容，每个层可以包含升级、挑战、成就等元素，开发者可以灵活组合这些模块。

**活跃的社区**：项目拥有超过 1,600 个 Fork，说明有大量开发者基于这个框架创建自己的游戏，社区生态非常活跃。

**详细的文档**：项目提供了完整的开发文档和教程，帮助新手快速上手。

### 适用场景

- 快速开发增量游戏原型
- 学习游戏框架的设计思路
- 了解模块化游戏开发方法
- 参与活跃的游戏开发社区

## 技术对比与选择建议

下表总结了四个项目的核心特征，帮助开发者根据自己的需求选择合适的参考项目：

| 项目 | 复杂度 | 代码量 | 学习曲线 | 适合场景 |
|------|--------|--------|----------|----------|
| **Antimatter Dimensions** | ⭐⭐⭐⭐⭐ | 超大 | 陡峭 | 学习复杂系统设计、大数处理 |
| **Kittens Game** | ⭐⭐⭐⭐ | 巨大 | 中等 | 学习资源管理、长期项目维护 |
| **The Modding Tree** | ⭐⭐ | 中等 | 平缓 | 快速开发、框架学习 |
| **Incremental Mass** | ⭐⭐⭐ | 中等 | 中等 | 现代设计、敏捷开发 |

### 选择建议

**如果你想学习复杂系统设计**：选择 **Antimatter Dimensions**，它展示了如何构建多层次的游戏系统和处理超大数值。

**如果你想学习资源管理**：选择 **Kittens Game**，它的资源系统设计堪称教科书级别。

**如果你想快速开发游戏**：选择 **The Modding Tree**，它提供了完整的框架和工具链，可以大幅缩短开发时间。

**如果你想了解现代设计趋势**：选择 **Incremental Mass Rewritten**，它代表了当前增量游戏的发展方向。

## 学习路径建议

对于想要深入学习增量游戏开发的开发者，建议按以下路径进行：

**第一阶段：快速入门**（1-2 周）

从 **The Modding Tree** 开始，使用框架快速搭建一个简单的增量游戏原型。这个阶段的重点是理解增量游戏的基本概念和核心玩法循环。

**第二阶段：深入理解**（2-4 周）

研究 **Incremental Mass Rewritten** 的源码，学习现代增量游戏的设计模式和代码组织方式。尝试为自己的原型添加更多功能和优化用户体验。

**第三阶段：系统设计**（4-8 周）

深入学习 **Kittens Game** 的资源管理系统，理解如何设计复杂的资源转化链和平衡多个系统之间的关系。

**第四阶段：高级技术**（8+ 周）

研究 **Antimatter Dimensions** 的大数处理和复杂系统架构，学习如何处理极端情况和优化性能。
